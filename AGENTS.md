# AGENTS.md

このリポジトリで作業する AI エージェント向けのガイドです。開発者向けチートシート集として、短く正確で再利用しやすい Markdown を保つことを優先してください。

## 作業方針

- 既存の構成と文体に合わせて、必要最小限の変更に留める
- チートシートは実用性を優先し、説明よりも使える例を中心にする
- コマンド、設定、コード例は動作前提が分かるようにする
- ソースコード例には、直前または直後に短い説明を添える
- コード内コメントは、初心者がつまずきやすい意図や注意点に絞って入れる
- バージョン依存、OS 依存、破壊的操作がある場合は明記する
- 一次情報や公式ドキュメントがある場合は参考リンクを添える

## Markdown ルール

- 見出しは `#`, `##`, `###` を使い、階層を深くしすぎない
- コマンド例には言語タグを付ける。例: `sh`, `powershell`, `bash`, `json`, `yaml`
- 長文よりも箇条書き、表、短いコードブロックを優先する
- コード例の説明は軽く書き、コードと同じ内容をなぞるだけのコメントは避ける
- ファイル名は原則として小文字の kebab-case にする。例: `docker-compose.md`
- 日本語の本文を基本にし、コマンド名や技術用語は一般的な表記を使う

## ディレクトリ方針

カテゴリが増えたら、次のような単位で整理してください。

- `git/`: Git, GitHub, ブランチ運用
- `shell/`: Bash, PowerShell, CLI 操作
- `docker/`: Docker, Docker Compose, コンテナ運用
- `mysql/`: MySQL, SQL, スキーマ設計、クエリ確認
- `javascript/`: JavaScript, TypeScript, Node.js
- `typescript/`: TypeScript, 型定義、tsconfig
- `python/`: Python, pip, venv, pytest
- `php/`: PHP, Composer, PDO
- `react/`: React, TypeScript, TSX, Hooks
- `java/`: Java, JVM, Maven, Gradle
- `security/`: セキュリティ確認、脆弱性調査、運用注意
- `tools/`: エディタ、開発支援ツール、汎用 CLI

新しい分類が必要な場合は、README の推奨構成も合わせて更新してください。

## 分野別ルール

- `react/` のコード例は TypeScript/TSX で書く
- React の Props は `type XxxProps` で明示し、必要に応じてイベント型や state の型引数も書く
- React のコードブロックには原則 `tsx` の言語タグを使う

## 品質チェック

変更後は可能な範囲で次を確認してください。

- Markdown の見出し階層が自然か
- コードブロックの言語タグが適切か
- ソースコード例に短い説明があり、必要な箇所だけコメントが入っているか
- コピペ可能なコマンドに危険な副作用がないか
- 古くなりやすい情報にバージョンや確認日があるか
- README からリポジトリの目的がすぐ分かるか

## 禁止事項

- API キー、トークン、パスワード、個人情報を追加しない
- 未確認の手順を断定的に書かない
- 既存ファイルの無関係な整形や大規模な書き換えをしない
- 破壊的なコマンドを警告なしで掲載しない
